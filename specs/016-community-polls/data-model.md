# Data Model: Community Polls & Voting — Client UI

**Branch**: `016-community-polls` | **Date**: 2026-03-03

## Overview

The client data model mirrors the server's GraphQL schema for polls. All types are derived from the generated GraphQL schema (after `pnpm codegen`). This document describes the client-side view models and form models that wrap the generated types.

---

## Server Entities (from GraphQL Schema)

These types are generated by codegen and consumed by the client. Documented here for reference.

### Poll

| Field                 | Type                 | Nullable | Description                                     |
| --------------------- | -------------------- | -------- | ----------------------------------------------- |
| id                    | UUID                 | No       | Unique identifier                               |

## Client View Models

These are the client-side types used in components. They extend or wrap the generated GraphQL types.

### CalloutDetailsModel Extension

The existing `CalloutDetailsModel.framing` gains a new optional `poll` field:

```
CalloutDetailsModel.framing
  ├── profile: ProfileModel
  ├── type: CalloutFramingType      (now includes POLL)
  ├── whiteboard?: WhiteboardDetails
  ├── memo?: MemoModel
  ├── link?: LinkDetails
  ├── mediaGallery?: MediaGalleryModel
  └── poll?: PollDetailsModel        ← NEW
```

### PollSettingsModel

| Field             | Type                  | Description                        |
| ----------------- | --------------------- | ---------------------------------- |

## State Transitions

### Poll Status

```
OPEN ──(future: close)──→ CLOSED
```

Only OPEN is implemented in this iteration. CLOSED is reserved.

### Vote Lifecycle (per user per poll)

```
No Vote ──(castPollVote)──→ Voted
Voted ──(castPollVote)──→ Voted (replaced)
Voted ──(option removed/edited by admin)──→ No Vote (server deletes vote)
```

### Poll UI States (per viewer)

```
┌─────────────────────┐
│  Has user voted?     │
├──────┬──────────────┤
│  No  │     Yes      │
├──────┼──────────────┤
│ Selection Mode       │ Results Mode        │
│ • Options displayed  │ • Options ranked    │
│ • Vote controls      │ • Results shown     │
│ • No results*        │ • "Change Vote" btn │
│ • Submit button      │ • User selections   │
│                      │   highlighted       │
└──────┴──────────────┘

* Unless resultsVisibility = VISIBLE, in which case
  results are shown alongside vote controls.
```

## Validation Rules

### Poll Creation

- Title: non-empty, max 512 characters
- Options: minimum 2, each non-empty, max 512 characters
- minResponses: ≥ 1
- maxResponses: ≥ 0; when > 0, must be ≥ minResponses

### Vote Submission

- selectedOptionIDs count must be ≥ poll.settings.minResponses
- selectedOptionIDs count must be ≤ poll.settings.maxResponses (when maxResponses > 0)
- All selectedOptionIDs must belong to the poll

### Option Management

- Add: text non-empty, max 512 characters
- Edit: text non-empty, max 512 characters
- Remove: poll must retain at least 2 options after removal
- Reorder: must include exactly all current option IDs
