# Implementation Plan: PWA Push Notifications

**Branch**: `023-pwa-push-notifications` | **Date**: 2026-03-16 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/023-pwa-push-notifications/spec.md`

## Summary

Add browser push notification support to the Alkemio client-web application. The server-side infrastructure (GraphQL API, VAPID auth, subscription storage, delivery pipeline) is fully implemented. This plan covers extending the existing service worker with push event handlers, creating a push subscription management hook, extending the notification settings UI from dual-toggle (email/inApp) to triple-toggle (email/inApp/push), and adding device management for multi-browser subscriptions.

## Technical Context

**Language/Version**: TypeScript 5.x, React 19
**Primary Dependencies**: Apollo Client (GraphQL), MUI 6 (UI), react-i18next (i18n), Vite (build)
**Storage**: Apollo InMemoryCache (client-side); server handles persistent subscription storage
**Testing**: Vitest with jsdom environment, SWC transforms
**Target Platform**: Modern browsers with Push API support (Chrome 50+, Firefox 44+, Edge 17+, Safari 16.1+)
**Project Type**: Web (SPA)
**Performance Goals**: < 20 KB gzipped added to main bundle; push notification delivery < 5s p95 (server-dependent)
**Constraints**: No new npm dependencies; native Push API + Service Worker API only; permission prompt only from settings page (never auto-prompt)
**Scale/Scope**: ~18k modules; changes touch ~15 files; 3 new GraphQL documents; 1 new hook; 1 new UI component; 1 service worker extension
