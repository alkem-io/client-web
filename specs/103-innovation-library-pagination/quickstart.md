# Quickstart: Paginated Innovation Library (Client Adoption)

**Feature**: 103-innovation-library-pagination

## Prerequisite — operation-hook codegen against a server-101 backend

The server-101 **schema types have already landed** in the client's generated
schema (`Library.templatesPaginated`, `Library.innovationPacksPaginated`, and
their result/filter types are present). What's still missing are the two
operation **hooks**. Generating them needs a backend at `localhost:3000/graphql`
exposing those fields (server feature `101-innovation-library-pagination`).

```bash
# With the server-101 backend running locally, after authoring the two .graphql docs:
pnpm codegen          # generates the operation hooks in src/core/apollo/generated/apollo-hooks.ts (commit the result)
```

Verify the schema fields are present (already true on this branch):

```bash
grep -n "templatesPaginated\|innovationPacksPaginated" src/core/apollo/generated/graphql-schema.ts
```

## Build / test the change

```bash
pnpm lint                                   # TypeScript + Biome + ESLint (react-compiler)
pnpm vitest run src/main/crdPages/innovationLibrary --reporter=basic
pnpm start                                  # dev server on localhost:3001
```

Open the CRD page (seed the design toggle if needed):

```js
localStorage.setItem('alkemio-design-version', '2');  // CRD
location.reload();
// then navigate to /innovation-library
```

## Manual acceptance walkthrough (maps to spec scenarios)

1. **First page bounded (US1.1, SC-001/002)** — On load, the Templates and
   Innovation Packs sections each show at most 25 items and a total count.
   DevTools → Network shows the two bounded paginated requests and **no** legacy
   unbounded `InnovationLibrary` request (SC-005).
2. **Load More appends (US1.2 / US3.2)** — Click "Load More" under Templates; the
   next ≤25 templates append below the existing ones (no flicker, no duplicates).
   Repeat for Packs.
3. **Last page hides control (US1.3 / US3.3, FR-008)** — Keep loading until the
   total is reached; the "Load More" control disappears.
4. **Server-side type filter (US2, SC-004)** — Select a template type; the total
   and the first page reflect only that type; paging stays within the filter.
   Change/clear the filter and confirm the list resets to the first page.
5. **Empty filtered set (US2.3, FR-013)** — Pick a type with no templates; the
   empty-state shows, total reads 0, no "Load More".
6. **Preview preserved (US-FR-010, SC-006)** — After loading several pages, click
   a template card; the `TemplatePreviewDialog` opens with content; closing it
   does not reset paging.
7. **Stale cursor recovery (FR-014)** — (Harder to force manually.) If an item is
   removed mid-browse and "Load More" hits a stale cursor, the section silently
   reloads its first page rather than erroring. Covered by a unit test that mocks
   a `fetchMore` rejection and asserts a first-page refetch.
