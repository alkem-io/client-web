### i18n guidelines (legacy core translations)

> **This directory is legacy and frozen for new keys.** It holds the MUI app's
> translations only. **New user-facing strings go to the CRD per-feature namespaces
> under `src/crd/i18n/<feature>/`** (all six languages — en, nl, es, bg, de, fr —
> edited in the same PR; managed manually, not Crowdin). See `src/crd/CLAUDE.md`.

For legacy upkeep of strings that already exist here:

- The project uses Crowdin for this directory only.
- In any PR make sure you change only the `translation.en.json` file and not any of the others.
- Do not add NEW keys here — add them to CRD instead.
- The other translations are updated through Crowdin.
- If you need to do a change in a non-en file, do it from Crowdin.
